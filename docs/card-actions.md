# Модульные действия карточек

Карточка должна описывать не только текст, но и структурированные действия в `card_effects`.
Backend читает эти действия и выполняет их без отдельного кода под конкретную карточку.

## Action types

| effectType | Что делает |
| --- | --- |
| `cash.adjust` | Меняет наличные игрока. Отрицательная сумма = обязательный расход, положительная = доход. |
| `cashflow.adjust` | Меняет базовый денежный поток: положительная сумма добавляет пассивный доход, отрицательная добавляет ежемесячный расход. |
| `liability.create` | Создаёт долг/пассив игрока и ежемесячный платёж. |
| `asset.quantity.multiply` | Умножает количество акций у всех владельцев указанного `symbol`. |
| `asset.quantity.divide` | Делит количество акций у всех владельцев указанного `symbol`. |
| `asset.wipeout` | Обнуляет активы с указанным `symbol`. |

Старые `effectType` остаются совместимыми:
`cash_delta`, `conditional_cash_delta`, `cashflow_delta`, `stock_split`,
`stock_reverse_split`, `stock_wipeout`.

## Обязательные расходы

Карточки типа `DOODAD` применяются автоматически. Игрок не может отказаться от оплаты,
если условие карточки выполнено.

Пример обязательного расхода:

```json
{
  "effectType": "cash.adjust",
  "amountCents": -1000,
  "payload": { "mandatory": true }
}
```

Пример обязательного долга:

```json
{
  "effectType": "liability.create",
  "amountCents": 17000,
  "payload": {
    "mandatory": true,
    "type": "doodad_loan",
    "name": "Новый водный глиссер",
    "paymentCents": 340
  }
}
```

## Условия

Условия хранятся в `card_conditions`.

Поддерживаются:

| condType | Что проверяет |
| --- | --- |
| `has_children` | У игрока есть хотя бы один ребёнок. |
| `has_rental_realestate` | У игрока есть сдаваемая недвижимость. |
| `has_8_plex` | У игрока есть 8-квартирный дом. |

Если условие не выполнено, действие не применяется, а в журнал пишется событие
`card:condition_not_met`.

## Покупаемые сделки

Для `SMALL_DEAL` и `BIG_DEAL` эффекты `cash.adjust` / `cash_delta` и
`cashflow.adjust` / `cashflow_delta` используются как параметры покупки.
Игрок может отказаться от такой сделки.

Для автоматического действия на карточке сделки нужно добавить в payload:

```json
{ "automatic": true }
```
